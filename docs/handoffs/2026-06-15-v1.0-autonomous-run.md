## How it was driven (reusable harness)

- **Orchestration loop:** `Workflow` runs each wave in the background and re-invokes the orchestrator on
  completion (event-driven — no polling). Per wave: compose a file-disjoint task set (≤1 task per hotspot
  file: `fp-app/main.rs`, `fp-app/screens.rs`, `fp-engine/lib.rs`, `fp-character/executor.rs`) → run the
  pipeline → merge green PRs (`gh pr merge --squash --admin`) → `git reset --hard origin/main` →
  `cargo test --workspace` + clippy + fmt integration gate → `fakoli-state` submit+apply → next wave.
- **Wave workflow script:** `.fakoli-state/wave-workflow.js` (gitignored). Args = `{repo, base, tasks[]}`
  with each task carrying its full spec inline (id, title, description, acceptance[], verify[],
  likely_files[]). Final-review script: `.fakoli-state/final-review.js`.
- **Combining tightly-coupled tasks:** when several ready tasks all hit the same hotspot file (the whole
  front-end shares `fp-app/main.rs`/`screens.rs`), bundle them into ONE agent/PR (learning #7) — collapsed
  ~6 serial front-end waves into ~2. Force-claim the conflict-group siblings via
  `fakoli-state claim TXXX --force` and submit all of them against the one PR.

## Gotchas / invariants learned this run (don't relearn)

1. **The integration gate is non-negotiable and catches what per-crate tests can't.** Agents/reviewers run
   crate-scoped tests (fast — skip SDL2/wgpu); a combined front-end PR edited `assets/data/system.def`
   (added the SETUP menu item) and broke an **`fp-ui`** integration test that only the `cargo test
   --workspace` gate runs. Always run the full-workspace gate on warm `main` after every wave; fix-forward
   (don't revert) when it's a stale downstream assertion.
2. **"All green" ≠ correct — the adversarial reviewer earns its keep.** T043's discovered-character path
   bug (a discovered char showed in the roster but failed to load on a *relative* dir arg) passed all unit
   tests because the test used an absolute temp dir; only a reviewer reasoning about the realistic
   `fp-app chars/` invocation caught it. Live validation then caught the game-root gap that even the
   reviewers missed. **Test absolute AND relative paths; validate the user's actual described workflow.**
3. **Review agent must have Bash to review a PR branch.** The peer reviewer was initially
   `agentType: 'fakoli-crew:critic'`, which has no Bash — it couldn't `gh pr diff` and silently reviewed
   the stale `main` checkout (a bogus T027 MUST_FIX exposed this). Fixed by dropping the agentType so both
   reviewers are full-tools agents with distinct lenses. The reliable guardrails throughout were
   **reviewer-1 (full tools) + the integration gate**; the comprehensive finale review (full tools, 0
   MUST_FIX) confirmed nothing slipped.
4. **`discover_chars(dir)` accepts a game root OR the `chars/` dir** (resolves `dir/chars/` if present).
   Same for `discover_motifs` → `dir/data/`. Keep that dual behavior.
5. **`git switch main` mid-orchestration transiently reverts working-tree files** to `main`'s version
   (edits live on the fix branch until merged). The `reset --hard origin/main` after the merge restores
   them. Don't be alarmed by a "file reverted" reminder during a merge+gate background command.
6. Pre-existing carry-overs still true: SFF v2 palettes are RGBA quads; no per-tick `warn!` in the 60 Hz
   loop; clean-room = synthetic fixtures only; `fakoli-state` is `record_only` (agents own git).

## Remaining non-blocking follow-ups (from the finale review; tracked in known-issues)

- **`stages/` under a game-root arg is not yet auto-discovered.** `chars/` and `data/` game-root
  resolution landed (PR #94); stage discovery still keys off the motif's `select.def` dir. Thread the
  directory arg into the stage scan for symmetry.

  only `ProjHit<id>`. Documented parity gap, never panics.
- **Mid-match snapshot/restore drops cosmetic explods/projectiles** (display-only entities are excluded
  from `MatchSnapshot` by design; replay-from-start is unaffected, so determinism holds).
- **`.snd` ADX is recognise-and-skip**, not decoded (T029). Decode is a future enhancement.
- No regression test yet for cross-entity redirected assignment (`enemy, var(n) := v`) timing.
