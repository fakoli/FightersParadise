# 08 — Faithfulness Audit (MUGEN-fidelity gap map)

Generated by the ultracode multi-agent audit workflow (8 subsystem finders → 8 adversarial
verifiers → synthesis). **65 gaps verified** (false positives removed by independent verifiers);
deduped into the **39-item prioritized roadmap** below. Drive tasks top-down; re-audit periodically.
This complements [06-execution-plan.md](06-execution-plan.md) (the live task ledger) — as items
land, mark them here and in the ledger.

## Summary
After deduplicating the 65 verified gaps into ~39 distinct work items, one architectural fact dominates the ranking: fp-engine::resolve_attack already holds BOTH characters and both AIR tables (engine lib.rs:692-708), but Character::tick/tick_with (executor.rs:191-206) is single-entity, so every trigger and controller evaluates with no opponent handle and Character::redirect unconditionally returns None (lib.rs:1358-1361). Threading the opponent + stage into the tick context is THE keystone unlock that gates the entire cross-entity cluster (redirects, P2BodyDist/P2Dist, ScreenPos/BackEdgeDist, RoundState, throws/Target*/p1stateno, priority/trade) — it is plumbing, not new mechanics, so it ranks #1. The other big lever is a set of small high-impact 'wired-but-dead' fixes that each unblock a whole behavior with one arm or a few struct fields: SelfState (get-hit recovery can return to control), the unmapped forward/back/run/airjump velocity const members (movement currently jumps straight up), Const720p arithmetic, GetHitVar(animtype) population (every hit currently plays the Light reaction), the attack/defence damage multipliers, and the friction stop-floor threshold. Many duplicate finder gaps collapsed: VelMul (2==62), AssertSpecial (8==28==63-part), throws (5==14==23==64), Const720p (10==32), animtype (15-residual==20==24-block), the dropped-Statedef-header family (29/35/36/37/31/19), spark (21==34), stage (43==51, 45 fixture), FNT/text (42==53, 46 fixture). Asset-blocked items (no stage .def, no .fnt/font, no fight.def fixtures) are pushed down and flagged: stage rendering, FNT/text, fight.def lifebars, storyboards. The CI asset-gate gap (58) is a meta-multiplier: because test-assets is gitignored and CI has no fetch step, the entire real-KFM regression net runs as green no-ops, silently hiding regressions in every fix above — worth scheduling once the conformance fixture lands. Counts were sanity-checked against the source: several finder overcounts (VelMul 11->4/7, NotHitBy 17 total confirmed, get-hit knockback already applies directly so HitVel* controllers are lower-urgency than they look, GetHitVars mostly populated except animtype/hitcount/type).
